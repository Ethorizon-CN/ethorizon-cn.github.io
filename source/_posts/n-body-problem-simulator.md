## 实现

### 可视元素

#### 星体("S")
对于N体问题，首先需要有N个星体，即N个点并用S表示，具体数量应遵循自定义的原则，让用户自行调整。
首先应创建变量 $n$ 用以保存用户输入的星体数量，同时键入 $N_{um}=[1,2,\dots,n]$ 这一列表以备不时之需。此时Desmos会自动补全等差数列。

{% callout type="warning" title="注意变量定义域" icon="fa-solid fa-triangle-exclamation" %}
 应注意为所有元素设定定义域（包括范围、滑块步长等），避免数学错误。
 例：
 $$
 n=15
 $$
 $$
 {2}\le{n}\le{100}~步长:1
 $$
  ~~谁家好人设置-1颗星啊你说对吧~~ 

 注：由于Desmos限制列表长度最大为10000项，故星体数量不得超过100个，否则受力列表项数 $n^2$ 将超过10000项（这一点会在之后提及）。~~设100真的不卡吗~~
{% endcallout %}

此处的星体作为点阵当然需要设置坐标。

**任务一：找到合适的方法表示S**

笔者在编写此处时想到了以下方案：

{% tabs %}

 <!-- tab 方案一 -->
 很容易能够想到令 $S=[(1,2),(3,4),...]$ ，然而为每一项单独设置毕竟不现实，因而方案一pass。

 <!-- tab 方案二 -->
 为解决以上问题，我们需要为 $S$ 批量随机赋值，因此我们引入函数 $random()$ 和列表推导式 $for$ 。~~这玩意在py里面不是叫循环么~~

 {% folding title="Desmos函数random()用法简介" class="blue" open=true %}
 什么，你问我怎么用？字面意思，random随机数。不过有几点细节要注意：
 1.写 $random()$ 函数必须带上括号（不管里面有没有东西）。
 2.在该函数的括号中写入一个自然数（记为 $x$ ）可以生成一个含有x项不相同的随机数的列表。
 3.该函数生成的随机数范围为 $[0,1]$ 且不可调（因为这样已经图灵完备了）。如果要调整，参考以下写法：
  $设随机数的范围为[a,b]({a,b}\in{\mathbb{R}}),则r=a+random()(b-a)$ 。
 {% endfolding %}

 {% folding title="Desmos列表推导式for用法简介" class="blue" open=true %}
  利用 $for$ 可以将一个列表中的所有元素遍历并一一代入计算。例如输入 $sin(i)~for~i=[1,2,\dots,10]$ 则会输出：
  =`0.0174524064373` `0.0348994967025` `0.0523359562429` `0.0697564737441` `0.0871557427477` `0.104528463268` `0.121869343405` `0.13917310096` `0.15643446504` `0.173648177667`
  如果循环的变量有2个则可以达到“双重循环”的效果（等同于循环中再嵌套一个循环。更多变量以此类推）：
   $(a,b)\space{for}\space{a}=[1,2,\dots,10],b=[1,2,\dots,10]$ 
  上例将会输出一个有 ${10}\times{10}=100$ 个点的列表。
 {% endfolding %}

 根据以上内容便可列出 $S=(a,b)~for~a=random(n),b=random(n)$ 。然而当你真的输入这个式子你会发现屏幕上有一大堆不明所以的点阵（如下图）。
 ![使用上述表达式产生的错误点阵，此时n=5。](/images/QQ_1783412946814.png)
 由于Desmos对于多重循环采用排列组合的逻辑，故该表达式输出的结果是一个含有 $n^{2}$ 个元素的列表，而我们实际需要的是n个元素的列表，故方案二pass。

 <!-- tab 方案三 -->

 {% folding title="Desmos赋值符 $\\to$ 用法简介" class="blue" open=true %}
  简单来说：使变量变成你指定的东西。
  例： $令a=0$ ，此时输入 ${a}\to{1}$ ，表达式左侧就会出现一个箭头。点击箭头即可完成赋值，这样的一次赋值在Desmos中称为操作（Actions）。如在本例中，如果不停触发这个操作，a的值则是： ${0}\to{1}\to{1}\to{1}\dots$ 
  同理可得： $令a=0,A={a}\to{a+1}$ 此时触发A，a的值就是： ${0}\to{1}\to{2}\to{3}\dots$ 。
 {% endfolding %}

 经过测试，把点的 $x$ 和 $y$ 坐标分别创建一个列表可以完美解决方案二中的问题，同时也方便了后面受力分析与计算，即：
  $令x_0=[],y_0=[],R_{esetCoordinates}=x_{0}\to{r_{ad}{random(n)-\frac{r_{ad}}{2}}},y_{0}\to{r_{ad}{random(n)-\frac{r_{ad}}{2}}},其中r_{ad}为星体随机位置半径$ 
  $x_{0}$ ， $y_{0}$ 都是空列表，在触发 $R_{esetCoordinates}$ 之后就会变为n个随机数。

{% endtabs %}

 综上，方案三成功 ~~（无感情~~ 

#### 星体属性("m"," $v_x$ "," $v_y$ ")
 作为恒星而非光子，星体总是有内禀质量，因而需要创建集合 $m$ 保存每个星体的质量用于计算。m需提前赋值，此处不做赘述。如：
 $$
 R_{esetMass}=m\to10{random(n)}
 $$
  $v_x$ 和 $v_y$ 用以保存恒星运动过程中的速度分量，创建为空集即可，运行时会自动写入数据。
